① HTML — la structure ▸

HTML = le squelette de la page : il décrit le contenu (titres, textes, images, liens), pas l'apparence.

Une balise s'ouvre et se ferme : <p>texte</p>. Certaines sont seules : <img>, <br>.

BaliseRôle
<h1>…<h6>Titres (h1 = le plus gros)
<p>Paragraphe
<a href="…">Lien (href = la destination)
<img src="…" alt="…">Image (src = fichier, alt = secours)
<ul> <li>Liste à puces / un élément
<form> <input>Formulaire / champ de saisie

Structure minimale d'une page :

<!DOCTYPE html>
<html>
  <head> <!-- infos : titre, CSS -->
    <title>Mon site</title>
  </head>
  <body> <!-- le contenu visible -->
    <h1>Bienvenue</h1>
  </body>
</html>
🎤 À dire à l'oral
« HTML structure le contenu, CSS gère l'apparence, et PHP ajoute la logique côté serveur. »
② CSS — l'apparence ▸

CSS = le style : couleurs, polices, tailles, positions. On cible des éléments avec des sélecteurs :

SélecteurCible
ptoutes les balises <p>
.annonceles éléments de classe "annonce" (point)
#menul'élément d'id "menu" (dièse, unique)
.annonce {
  color: white;          /* couleur du texte */
  background-color: navy;/* fond */
  text-align: center;    /* centrage */
  padding: 10px;         /* marge intérieure */
}
⚠️ Piège fréquent
Chaque ligne CSS finit par un point-virgule ;. color = texte, background-color = fond (à ne pas confondre).
💡 Mémo
point = classe (réutilisable sur plusieurs éléments) · dièse = id (un seul élément).
③ PHP — la logique serveur ▸

PHP s'exécute sur le serveur et génère du HTML. Le code va entre <?php … ?>.

  • Variable : commence par $ → $prix = 50;
  • Afficher : echo "Bonjour";
  • Concaténer (coller) : avec un point → echo "Prix : " . $prix;

Conditions & boucles :

if ($prix > 100) {
  echo "Cher";
} else {
  echo "Abordable";
}

for ($i = 0; $i < 5; $i++) {
  echo $i;   // 0,1,2,3,4
}

Récupérer un formulaire :

<form method="post">
  <input name="titre">
</form>

// côté PHP :
$t = $_POST['titre'];   // POST = caché
$q = $_GET['recherche'];// GET = dans l'URL
🎤 À dire à l'oral (projet Leboncoin)
« L'utilisateur remplit un formulaire HTML, PHP récupère les données avec $_POST, les enregistre, puis réaffiche les annonces. »
💡 POST vs GET
POST = données cachées (mot de passe, formulaire). GET = données visibles dans l'URL (recherche, filtre).
① Variables & types ▸

Une variable stocke une valeur. Pas besoin de $ ni de déclarer le type, Python le devine.

nom = "Léonard"   # str  (texte)
age = 18          # int  (entier)
taille = 1.80     # float (décimal)
majeur = True     # bool (Vrai/Faux)
print(nom, age)
TypeExemple
str"bonjour" (entre guillemets)
int42
float3.14
boolTrue / False
⚠️ = ou == ?
= donne une valeur (age = 18). == compare (if age == 18).
② Conditions & boucles ▸

Python n'utilise pas d'accolades : c'est l'indentation (le décalage de 4 espaces) qui délimite les blocs.

if age >= 18:
    print("Majeur")
elif age > 0:
    print("Mineur")
else:
    print("Erreur")
for i in range(5):   # 0,1,2,3,4
    print(i)

x = 0
while x < 3:         # tant que x < 3
    print(x)
    x = x + 1        # sinon boucle infinie !
⚠️ Erreur n°1 du débutant
Oublier le : en fin de ligne ou ne pas indenter → IndentationError. Le : annonce un bloc, l'indentation = le contenu du bloc.
③ Listes, dictionnaires & fonctions ▸

Liste = plusieurs valeurs ordonnées (les index commencent à 0) :

contacts = ["Léo", "Francis", "Marie"]
print(contacts[0])   # Léo
print(contacts[-1])  # Marie (dernier)
contacts.append("Paul")  # ajoute

Dictionnaire = des paires clé → valeur (idéal pour un contact) :

contact = {"nom": "Léo", "tel": "0612345678"}
print(contact["nom"])  # Léo

Fonction = un bloc réutilisable :

def saluer(nom):
    return "Salut " + nom

print(saluer("Léo"))  # Salut Léo

nom = input("Ton nom : ")  # demande à l'utilisateur

① Le modèle OSI — les 7 couches ▸

Le modèle OSI découpe la communication réseau en 7 couches, chacune avec un rôle précis. La donnée descend les couches à l'envoi, et les remonte à la réception.

7
ApplicationCe que tu utilises : navigateur, mail (HTTP, SMTP)
6
PrésentationFormat, chiffrement, compression des données
5
SessionOuvre / maintient / ferme le dialogue
4
TransportDécoupe en segments — TCP (fiable) / UDP (rapide)
3
RéseauAdressage IP & routage entre réseaux
2
LiaisonAdresse MAC, réseau local (switch)
1
PhysiqueBits, signaux, câble / fibre / wifi
💡 Moyen mnémotechnique (bas → haut)
Pour Le Réseau, Tout Se Passe Automatiquement
= Physique · Liaison · Réseau · Transport · Session · Présentation · Application
② Qui travaille à quelle couche ? ▸
ÉlémentCouche
HTTP, HTTPS, mail7 — Application
TCP / UDP4 — Transport
Adresse IP, routeur3 — Réseau
Adresse MAC, switch2 — Liaison
Câble, fibre, wifi1 — Physique
💡 Les pièges à retenir
MAC = couche 2 (locale, gravée dans la carte) · IP = couche 3 (logique, mondiale).
Switch = couche 2 · Routeur = couche 3 (Routeur → Réseau).
⚠️ TCP vs UDP
TCP = fiable, vérifie que tout arrive (web, mail). UDP = rapide mais sans garantie (jeux, streaming, appels).
③ Notions clés à maîtriser ▸
  • Encapsulation : à l'envoi, chaque couche ajoute son en-tête autour de la donnée (comme des poupées russes). À la réception → désencapsulation.
  • Adresse IP : identifie une machine sur le réseau (ex. 192.168.1.10).
  • Adresse MAC : identifiant matériel unique de la carte réseau.
  • LAN : réseau local (maison, école). WAN : réseau étendu (Internet).
🎤 LA phrase à dire à l'oral
« L'intérêt du modèle en couches, c'est que chaque couche a un rôle unique et peut évoluer indépendamment : le wifi a remplacé le câble en couche 1 sans rien changer aux couches du dessus. »
