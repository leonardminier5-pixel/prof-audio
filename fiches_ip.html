1 Les bases : IP, octets, binaire ▸

Une adresse IPv4 = 4 octets séparés par des points : 192.168.0.10. Chaque octet va de 0 à 255 (8 bits) → l'adresse fait 32 bits au total.

Un octet en binaire = 8 cases, chacune valant une puissance de 2 :

2⁷2⁶2⁵2⁴2³2²2¹2⁰
1286432168421

On additionne les cases à 1. Ex : 11000000 = 128 + 64 = 192. Et 11111111 = 255.

💡 À retenir par cœur
La suite des 128 / 64 / 32 / 16 / 8 / 4 / 2 / 1 (on divise par 2 à chaque fois). C'est la base de TOUT le chapitre.
2 Le masque & les 8 valeurs ▸

Un masque de sous-réseau sépare l'adresse en 2 parties : la partie réseau (les bits à 1) et la partie machine (les bits à 0). Dans un masque, les 1 sont toujours collés à gauche, puis les 0.

Du coup, un octet de masque ne peut prendre que 8 valeurs possibles (on remplit les 1 de gauche à droite) :

Bits à 1BinaireValeur

411110000240

811111111255
💡 Mémo des valeurs
128 · 192 · 224 · 240 · 248 · 252 · 254 · 255
Une valeur de masque hors de cette liste (ex. 200, 130…) est impossible.
3 Convertir CIDR ⇄ masque ▸

La notation CIDR /n = le nombre de bits à 1 dans le masque (de gauche à droite).

/24  → 8+8+8+0 bits → 255.255.255.0
/28  → 8+8+8+4 bits → 255.255.255.240   (4 bits = 240)
/19  → 8+8+3+0 bits → 255.255.224.0     (3 bits = 224)
/13  → 8+5+0+0 bits → 255.248.0.0

Méthode : tu remplis les octets de 8 en 8. L'octet « à cheval » prend le reste via le tableau des 8 valeurs.

⚠️ Masques impossibles
  • 255.254.128.0 → les 1 ne sont pas continus (…1110.1000…) → alternance interdite.
  • 255.0.255.0 → un trou au milieu → impossible.
  • 255.256.240.0 → 256 n'existe pas (max = 255).
  • 255.0.0 → seulement 3 octets, il en faut 4.

Table de référence (octet qui varie + nb de machines) :

CIDRMasqueBlocMachines utiles

« Bloc » = taille d'un sous-réseau dans l'octet concerné = le nombre magique (voir section 4).

4 Trouver l'adresse réseau & le broadcast ▸

C'est l'exercice clé. On te donne une IP + un CIDR, tu dois trouver l'adresse réseau et l'adresse de diffusion (broadcast).

🔑 Le nombre magique
Nombre magique = 256 − (valeur du masque dans l'octet à cheval)
C'est la taille d'un bloc. Les adresses réseau sont toujours des multiples de ce nombre.
1
Convertis le CIDR en masque.
2
Repère l'octet où commence le premier 0 du masque = l'octet de travail.
3
Calcule le nombre magique = 256 − valeur du masque dans cet octet.
4
Trouve le plus grand multiple de ce nombre ≤ à la valeur de l'octet dans l'IP → c'est l'adresse réseau.
5
Multiple suivant − 1 = broadcast. Les octets après celui de travail : 0 pour le réseau, 255 pour le broadcast.

Exemple A — 172.17.24.171 /28

/28 → masque 255.255.255.240   octet de travail = 4e (171)
Nombre magique = 256 − 240 = 16
Multiples de 16 ≤ 171 :  10 × 16 = 160   (11×16 = 176 > 171)
→ Réseau    : 172.17.24.160 /28
→ Broadcast : 11×16 − 1 = 175 → 172.17.24.175 /28
  1re utilisable .161  ·  dernière utilisable .174  ·  14 machines

Exemple B — 192.168.211.112 /19 (l'octet de travail n'est pas le dernier !)

/19 → masque 255.255.224.0    octet de travail = 3e (211)
Nombre magique = 256 − 224 = 32
Multiples de 32 ≤ 211 :  6 × 32 = 192   (7×32 = 224 > 211)
→ Réseau    : 192.168.192.0 /19   (4e octet = 0)
→ Broadcast : 7×32 − 1 = 223 → 192.168.223.255 /19  (4e octet = 255)
💡 Vérif rapide
L'adresse réseau finit toujours par des 0 dans la partie machine, le broadcast par des 1 (255). Le broadcast = juste avant le réseau suivant.
5 Administrer des groupes (VLSM) ▸

On te donne une adresse de départ et plusieurs groupes avec un nombre de machines (postes) chacun. Il faut découper la plage pour loger tous les groupes sans qu'ils se chevauchent.

1
Pour chaque groupe : machines + 2 (on réserve l'adresse réseau + le broadcast).
2
Arrondis à la puissance de 2 supérieure ou égale (16, 32, 64, 128…). C'est la taille du bloc.
3
Masque = 32 − l'exposant. Ex : bloc 32 = 25 → /(32−5) = /27.
4
Place chaque groupe sur un multiple entier de sa taille de bloc, en évitant les zones déjà prises. Astuce : place les plus gros groupes en premier.
🔑 Rappel puissances de 2
2³=8 · 2⁴=16 · 2⁵=32 · 2⁶=64 · 2⁷=128 · 2⁸=256
bloc 8→/29 · 16→/28 · 32→/27 · 64→/26 · 128→/25 · 256→/24

Exemple complet — 192.168.0.0 /24 (256 adresses, dernier octet libre)

G1 = 24 postes → 24+2 = 26 ≤ 32 (2⁵) → /27 (bloc 32)
G2 = 14 postes → 14+2 = 16 ≤ 16 (2⁴) → /28 (bloc 16)
G3 = 57 postes → 57+2 = 59 ≤ 64 (2⁶) → /26 (bloc 64)
G4 = 63 postes → 63+2 = 65 ≤ 128(2⁷) → /25 (bloc 128)
G1 /27 (bloc 32)  →  0 × 32 = 0
   Réseau 192.168.0.0  ·  utilisables .1 → .30  ·  Broadcast .31
G2 /28 (bloc 16)  →  2 × 16 = 32   (.0–.31 pris par G1)
   Réseau 192.168.0.32 ·  utilisables .33 → .46 ·  Broadcast .47
G3 /26 (bloc 64)  →  1 × 64 = 64   (.48–.63 trop petit)
   Réseau 192.168.0.64 ·  utilisables .65 → .126 · Broadcast .127
G4 /25 (bloc 128) →  1 × 128 = 128
   Réseau 192.168.0.128· utilisables .129 → .254· Broadcast .255
⚠️ Les erreurs qui coûtent des points
  • Multiples ENTIERS uniquement. 0,5 × 32 = 16 → interdit. Un groupe /27 ne peut commencer qu'à 0, 32, 64, 96…
  • Pas de chevauchement : vérifie que le bloc ne mord pas sur un groupe déjà placé.
  • Ne pas oublier le +2 (réseau + broadcast) avant d'arrondir.
🎤 À dire à l'oral
« Je calcule la taille de chaque groupe (machines + 2), je l'arrondis à la puissance de 2 supérieure pour obtenir le masque, puis je place les blocs sur des multiples entiers de leur taille, du plus grand au plus petit, sans chevauchement. »
